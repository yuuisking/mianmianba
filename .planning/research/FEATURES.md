# V2 Feature Research

**Date:** 2026-05-13
**Scope:** 面面吧 V2 全产品能力范围

## Table Stakes

### 候选画像与岗位建模

- 用户可输入目标公司、岗位、级别、JD、简历并生成统一候选画像。
- 系统可输出岗位能力权重、轮次预测、公司流程假设与面试关注点。

### 阶段面试

- 用户可发起单场阶段面试并获得岗位化首题与连续追问。
- 系统可在刷新/重进后恢复当前会话进度、消息、计时与报告中间态。

### 全流程面试

- 用户可查看面试计划列表、创建新流程、看到当前轮次、后续轮次与风险。
- 每个阶段都应有独立面试官角色、阶段目标、通过条件和过轮规则。

### 证据化评估与报告

- 每轮都要有证据抽取、维度评分、理由与置信度。
- 面试结束后必须先返回可读骨架报告，再补齐长文本与证据引用。
- 当证据不足时必须拒绝伪造正式结论。

### 纠偏闭环

- 复盘中心可识别稳定弱项、波动问题与已修复问题。
- 弱项可直接回流到专项训练、同题重答、学习卡与下一轮面试准备。

## Differentiators

### 多 Agent 评审团

- 用独立 Judge、校准、信任守门、评审投票替代“一个模型包打一切”。
- 支持全流程阶段过轮投票、borderline pass 与风险继承。

### 通用能力内核 + 岗位适配器

- 先沉淀结构化表达、证据、拆解、权衡、真实性等通用维度。
- 再通过岗位适配器配置能力权重、题型树、术语、案例与通过标准。

### 成长与 Readiness

- 成长档案持续跟踪“稳定提升 / 波动 / 回退”。
- Offer Readiness 直接回答“现在能不能投、最可能卡在哪一轮、最短补强路径是什么”。

### 面经智能化利用

- 面经不是展示素材，而是流程生成、轮次策略、题目聚焦和风险判断的真实输入。

## Anti-Features

- 把 V2 再做成“高级聊天页” —— 与目标护城河方向相反。
- 报告只输出漂亮总结，不输出原句证据与行动建议 —— 会导致系统不可验证。
- 没有状态机就直接堆功能入口 —— 近期线上混乱已经证明这条路不可持续。
- 一次性横向铺满所有岗位 —— 会迅速稀释评估标准和内容质量。

## Build Implications

- 先完成建模、状态机、评估内核，再接运行时与复盘闭环。
- 全流程、算法题、报告、复盘不能继续分散迭代，必须共享统一证据链与状态链。
- 成长档案与 readiness 必须建立在前面模块稳定可追溯的前提上，不能提前单独包装页面。
