<objective>
完成 `interview -> report` 主链收口与三层观测最小骨架，让报告不再是唯一状态收口点，并让用户主链、状态事件链、Agent 摘要链都能为排障提供统一入口。
</objective>

<must_haves>
<truths>
- D-14: 第二优先主链是 `interview -> report`。
- D-18: 三层观测都必须落地。
- D-19: Agent 运行链 Phase 1 先做摘要级。
- D-20: 线上问题默认第一排查入口是用户主链回放。
- D-21: 观测层目标是证明状态机、报告与迁移按设计运行，而不是堆日志。
</truths>
</must_haves>

<threat_model>
- T-13-HIGH: 如果报告生成继续顺带承担状态收口副作用，任何报告链故障都会同时破坏状态链。
- T-14-HIGH: 如果聊天主链不写 Agent 摘要链，后续出现“为什么这么问/为什么结束”时无法排障。
- T-15-MEDIUM: 如果继续依赖浏览器缓存作为 report fallback 主真相，用户主链回放就不可信。
</threat_model>
