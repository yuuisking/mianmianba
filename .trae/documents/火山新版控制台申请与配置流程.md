# 火山新版控制台申请与配置流程

> 历史说明：当前仓库主方案已调整为“`Qwen3-TTS` 负责语音合成，火山仅保留 `ASR`”。本文不再作为当前 `TTS` 主申请文档，仅用于保留火山语音识别配置时参考。

本文面向当前 `resumer` 项目的实时语音链路，目标是让你按新版豆包语音控制台的最短路径完成申请、拿到凭证，并能在域名与 HTTPS 就绪后直接上线联调。

## 一、先说结论

当前项目已经完成以下改造：

- 本地 `ASR` 已改为火山官方 `WebSocket` 协议并真实打通。
- 代码已经兼容两套识别模式：
  - `legacy`：兼容旧版 `APP ID + Access Token + Cluster`
  - `bigmodel`：新版推荐，使用 `API Key + Resource ID`

当前最新边界是：

- 火山：只保留 `ASR`
- 阿里百炼：负责 `TTS`

因此你现在在火山侧的主要工作，不再是申请 `TTS` 相关资源，而是把正式生产要用的 `ASR` 参数申请齐。

---

## 二、你最终需要拿到什么

### 1. 识别相关

- `VOLC_API_KEY`
- `VOLC_ASR_RESOURCE_ID`

推荐值：

- `VOLC_ASR_RESOURCE_ID=volc.seedasr.sauc.duration`

如果你后续明确走并发包，也可以改成：

- `VOLC_ASR_RESOURCE_ID=volc.seedasr.sauc.concurrent`

### 2. 当前新版口径说明

本项目现已收敛到新版火山口径：

- `ASR` 使用 `API Key + ASR Resource ID`
- `TTS` 当前主方案已切换为阿里百炼 `Qwen3-TTS`

也就是说，**火山侧正式生产接入当前只关注 `ASR`，不再以火山 `TTS` 作为当前主路径**。

如果你希望保留旧版参数用于历史兼容或排障，可以额外保存，但当前新版主链路已经不再强依赖它们。

## 四、新版控制台详细申请步骤

### 第 1 步：进入新版豆包语音控制台

- 新版总览页：[https://console.volcengine.com/speech/new/overview?projectName=default](https://console.volcengine.com/speech/new/overview?projectName=default)
- 官方新版快速入门：[https://www.volcengine.com/docs/6561/2119699?lang=zh](https://www.volcengine.com/docs/6561/2119699?lang=zh)

你会看到新版概览页，里面会有“开通服务”“立即体验”“API接入”等入口。

### 第 2 步：一键开通新版语音服务

在新版概览页：

- 找到“开通服务”区域
- 点击“前往开通”或等价按钮
- 优先开通和本项目直接相关的能力

建议至少确认以下两类能力是可用状态：

- `豆包流式语音识别模型 2.0`
- `语音合成` 或 `豆包语音合成模型 2.0`

参考文档：

- 语音识别大模型产品简介：[https://www.volcengine.com/docs/6561/1354871?lang=zh](https://www.volcengine.com/docs/6561/1354871?lang=zh)
- 语音合成大模型产品简介：[https://www.volcengine.com/docs/6561/1257543](https://www.volcengine.com/docs/6561/1257543)

### 第 3 步：确认默认项目空间

新版控制台默认会落在 `default` 项目空间。

你需要确认：

- 当前页面右上或左上显示的项目空间是 `default`
- 后续申请的 `API Key` 与语音服务都在同一个项目空间下

否则你会遇到“明明创建了 API Key，但调用时权限不对”的问题。

### 第 4 步：创建新版 API Key

- 直接进入 API Key 页面：[https://console.volcengine.com/speech/new/setting/apikeys?projectName=default](https://console.volcengine.com/speech/new/setting/apikeys?projectName=default)
- 官方文档：[https://www.volcengine.com/docs/6561/1816214?lang=zh](https://www.volcengine.com/docs/6561/1816214?lang=zh)

操作建议：

1. 点击“创建 API Key”
2. 名称建议填写：`resumer-realtime-voice-prod`
3. 创建完成后立即复制保存

你拿到的值对应：

- `VOLC_API_KEY`

注意：

- `API Key` 很敏感，不能发到群里，不能提交到 Git。
- 如果怀疑泄露，可以在这个页面禁用或删除后重新创建。

### 第 5 步：确认流式语音识别 Resource ID

官方文档：

- 大模型流式语音识别 API：[https://www.volcengine.com/docs/6561/1354869?lang=zh](https://www.volcengine.com/docs/6561/1354869?lang=zh)

新版文档里明确给了可用 `Resource ID`：

- `volc.bigasr.sauc.duration`
- `volc.bigasr.sauc.concurrent`
- `volc.seedasr.sauc.duration`
- `volc.seedasr.sauc.concurrent`

本项目建议优先使用：

- `volc.seedasr.sauc.duration`

原因：

- 这是更贴近新版豆包流式识别 2.0 的推荐口径。
- 当前阶段我们更需要先稳定打通时长计费模式，而不是先做并发包采购。

你最终需要保存：

- `VOLC_ASR_RESOURCE_ID=volc.seedasr.sauc.duration`

### 第 6 步：确认 TTS Resource ID

新版 `TTS` 不再要求你去旧版找 `APP ID / Access Token / Cluster` 才能用。

官方文档：

- V3 HTTP Chunked 单向流式：[https://www.volcengine.com/docs/6561/1598757?lang=zh](https://www.volcengine.com/docs/6561/1598757?lang=zh)
- V3 WebSocket 单向流式：[https://www.volcengine.com/docs/6561/1719100?lang=zh](https://www.volcengine.com/docs/6561/1719100?lang=zh)
- V3 WebSocket 双向流式：[https://www.volcengine.com/docs/6561/1329505?lang=zh](https://www.volcengine.com/docs/6561/1329505?lang=zh)

文档里已经明确给出 `TTS Resource ID`：

- `seed-tts-2.0`
- `seed-tts-1.0`
- `seed-tts-1.0-concurr`

本项目当前建议固定使用：

- `VOLC_TTS_RESOURCE_ID=seed-tts-2.0`

原因：

- 你当前已经开通的是 `语音合成2.0`
- 我们的项目目标是尽量对齐豆包 2.0 语音体验
- 2.0 音色体系更适合后续继续做“低 AI 味”和更自然接话

### 第 7 步：确认语音合成音色

当前项目里，面试官播报仍依赖 `voice_type`。

你需要在控制台确认一个可长期使用的音色 ID。

建议：

- 先选择一个中文、稳定、偏自然的官方音色
- 不要一开始就上复杂情绪音色

最终要保存：

- `VOLC_TTS_VOICE_TYPE`

官方语音合成能力说明：

- [https://www.volcengine.com/docs/6561/1257543](https://www.volcengine.com/docs/6561/1257543)

文档页中有音色列表入口，可继续跳转查看。

### 第 8 步：确认试用与正式版状态

新版控制台虽然操作更简单，但有一个关键点不能忽略：

- 试用额度用完后，会自动转入后付费或无法继续使用，具体以控制台状态为准。

官方参考：

- 计费概述：[https://www.volcengine.com/docs/6561/1359369](https://www.volcengine.com/docs/6561/1359369)
- 计费说明：[https://www.volcengine.com/docs/6561/1359370?lang=zh](https://www.volcengine.com/docs/6561/1359370?lang=zh)

你需要至少确认三件事：

1. `豆包流式语音识别模型 2.0` 是否已开通
2. `语音合成` 或 `豆包语音合成模型 2.0` 是否已开通
3. 当前是试用态还是正式版

如果你准备域名一好就上线联调，我建议：

- 不要只依赖试用额度
- 提前完成正式开通或至少确认充值能力正常

### 第 9 步：确认并发和额度

官方文档中说明，流式识别正式版通常有默认并发限制，具体以控制台为准。

参考：

- [https://www.volcengine.com/docs/6561/1354871?lang=zh](https://www.volcengine.com/docs/6561/1354871?lang=zh)

你要确认：

- 默认并发是否够你现在内测使用
- 是否已经有资源包或后付费额度
- 是否需要后续扩容

如果只是当前项目阶段，先满足：

- 单人/小规模联调
- 多轮语音面试自测

通常默认并发就够。

## 五、申请完成后怎么填到项目里

### 1. 推荐环境变量模板

```bash
# 新版推荐参数
VOLC_API_KEY=你的_API_KEY
VOLC_ASR_RESOURCE_ID=volc.seedasr.sauc.duration
VOLC_TTS_RESOURCE_ID=seed-tts-2.0
VOLC_ASR_WS_MODE=bigmodel
VOLC_ASR_MODEL_NAME=bigmodel
VOLC_ASR_ENABLE_NONSTREAM=true
VOLC_ASR_END_WINDOW_SIZE=800
VOLC_ASR_FORCE_TO_SPEECH_TIME=1000

# TTS 参数
VOLC_TTS_VOICE_TYPE=你的_voice_type
VOLC_TTS_SPEED_RATIO=1.0
VOLC_TTS_PITCH_RATIO=1.0
VOLC_TTS_VOLUME_RATIO=1.0
```

### 2. 当前项目中的作用

- `VOLC_API_KEY`：新版大模型流式识别鉴权
- `VOLC_ASR_RESOURCE_ID`：新版大模型流式识别资源
- `VOLC_TTS_RESOURCE_ID`：新版语音合成 2.0 资源
- `VOLC_ASR_WS_MODE=bigmodel`：强制项目优先走新版大模型识别
- `VOLC_TTS_VOICE_TYPE`：控制面试官播报音色

## 六、申请后如何快速验证

当前仓库已经新增自检脚本：

- `scripts/volc-speech-smoke-test.mjs`

使用方式：

1. 先把新版参数配置到本地环境变量
2. 启动本地项目：

```bash
npm run dev
```

3. 另开一个终端运行：

```bash
node scripts/volc-speech-smoke-test.mjs
```

预期结果：

- `TTS` 通过并生成一条测试音频
- `ASR` 通过并返回中文识别结果

如果这一步通过，说明：

- 你的火山参数已经正确
- 当前项目已经能真正调用火山语音
- 后续主要剩下域名、HTTPS 和浏览器设备权限联调

## 八、我这边接下来怎么配合你

你一旦把新版参数申请下来，我会按下面顺序继续推进：

1. 切换项目到 `bigmodel` 正式模式
2. 再做一轮本地 `ASR + TTS` 实测
3. 收口房间级 partial/final 字幕
4. 收口用户打断与 AI 停播同步
5. 等域名和 HTTPS 就绪后，直接做浏览器真机联调

这样你的域名一就绪，不会再从头接火山，而是直接进入上线前最后一轮体验收口。
