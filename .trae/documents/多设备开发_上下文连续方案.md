# 多设备开发上下文连续方案（公司电脑 ↔ 家里电脑）

## Summary
目标是在公司电脑与家里电脑之间切换开发时，尽量做到“项目状态连续、上下文不丢失”。本方案采用 **GitHub 私有仓库作为主同步通道**，并用 **文件拷贝** 同步本地数据库与环境变量（你要求同步 `.env.local`）来保持两端一致。

## Current State Analysis（基于仓库实况）
- 项目根目录：`/Users/didi/workplace/project/resumer`
- Trae 的上下文资产已落盘在 `.trae/` 下：
  - `.trae/specs/`：各功能 Spec/Tasks/Checklist（可作为“长期上下文”）
  - `.trae/documents/`：已有文档（例如 `ui_redesign_plan.md`）
- 学习中心知识内容在 `data/learning-center.json`
- 本地数据库是 SQLite：`prisma/dev.db`
- `.gitignore` 当前忽略了 `.env*`（即 `.env.local` 默认不会进入 Git）
- 当前目录下未发现 `.git/`（尚未初始化 Git 或未在本目录保存）

## Key Idea：区分三类“上下文”
1. **代码上下文**：`src/`, `prisma/schema.prisma`, `package.json` 等 —— 用 GitHub 同步
2. **产品/决策上下文**：`.trae/specs/`, `.trae/documents/`, `AI模拟面试平台_PRD.md`, `UI/` —— 用 GitHub 同步（Trae Solo 在家里可沿着这些继续做）
3. **运行态上下文**：`data/learning-center.json`, `prisma/dev.db`, `.env.local` —— 建议用“拷贝文件/加密同步”补齐（你希望不丢失）

## Proposed Changes（你需要做的事，按顺序）

### A. 用 GitHub 私有仓库同步“代码 + 决策上下文”（推荐主路径）
1. **在公司电脑初始化并推送仓库**
   - 在项目根目录初始化 Git（若尚未初始化）
   - 创建 GitHub 私有仓库（Private）
   - 关联 remote 并 push
2. **确认将以下目录纳入版本管理（上下文来源）**
   - `.trae/specs/`
   - `.trae/documents/`
   - `UI/`
   - `AI模拟面试平台_PRD.md`
3. **建议新增一个“交接文档”（保持长期上下文）**
   - 文件建议：`.trae/documents/handoff.md`
   - 内容建议：当前在做什么、最近改动点、下一步做什么、重要路径/接口/环境变量清单
   - 目的：家里电脑的 Trae Solo 直接读这一份就能接续上下文

### B. 用“拷贝文件”同步运行态上下文（你要求同步 `.env.local`）
因为 `.gitignore` 会忽略 `.env*`（这是安全默认），而且 `.env.local` 属于敏感信息，不建议进入 GitHub，即使是私有仓库也有泄露风险。

你可选两种方式（按你的偏好“直接同步 .env.local”给出可执行做法）：
1. **方式 B1：手动拷贝（最简单）**
   - 每次切换电脑前，把以下文件拷到 U 盘/网盘：
     - `.env.local`
     - `data/learning-center.json`
     - `prisma/dev.db`
   - 家里电脑覆盖同名文件（保持一致）
2. **方式 B2：加密后再同步（更安全，推荐但需要一点工具）**
   - 用 1Password/Bitwarden 保存环境变量
   - 或用 zip + 密码（最小可行），同步加密压缩包

> 决策：你当前选择“直接同步 .env.local”，方案默认采用 B1；但强烈建议未来迁移到 B2。

### C. 家里电脑第一次接入（让 Trae Solo 能“沿着项目继续做”）
1. **拉取代码**
   - 从 GitHub clone 项目到家里电脑（路径随你）
2. **恢复运行态文件**
   - 把公司电脑的 `.env.local`、`data/learning-center.json`、`prisma/dev.db` 拷贝到家里电脑相同位置
3. **安装依赖并启动**
   - 安装 Node.js（建议与公司电脑同大版本）
   - `npm install`
   - 如 Prisma 需要：根据现状选择
     - 若同步了 `prisma/dev.db`：通常无需额外迁移即可跑
     - 若不同步 DB：需要 `prisma db push` 或迁移（后续再规范化）
4. **Trae Solo 的上下文接续方式**
   - 打开项目根目录，让 Trae 索引代码
   - 让 Trae 优先阅读：
     - `.trae/specs/`（正在推进/已完成的 Spec）
     - `.trae/documents/handoff.md`（若你新增）
     - `AI模拟面试平台_PRD.md`

## Assumptions & Decisions
- 你接受使用 GitHub（私有仓库）作为主要同步渠道
- 你希望“运行态上下文不丢失”，所以包含：`.env.local` + `data/learning-center.json` + `prisma/dev.db`
- `.env.local` 不进 Git（默认安全实践）；通过拷贝文件实现你想要的“直接同步”
