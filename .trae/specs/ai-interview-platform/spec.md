# AI 模拟面试平台 Spec

## Why
面试与求职训练需求长期存在，现有的刷题和纯聊天模式无法提供“真实面试节奏、追问深挖以及针对简历/JD的个性化反馈”。本平台旨在通过AI提供高度还原的模拟面试体验，输出结构化反馈报告，帮助求职者（特别是校招和1-3年经验者）提升面试能力。

## What Changes
- 初始化全栈项目：使用 Next.js (TypeScript) + Prisma + PostgreSQL 搭建底层架构。
- 100% 还原 UI：完全复用提供的 `index.html` 和 CSS 样式，将其拆分为 React 组件，禁止引入第三方 UI 库导致视觉偏离，确保视觉效果与设计稿完全一致。
- 用户认证模块：实现注册/登录体系（基于 NextAuth）。
- 简历与 JD 解析模块：集成 AI 接口，支持文本/文件输入，提取技能与经验，生成结构化候选人画像。
- 面试引擎模块：基于候选人画像和所选模板（校招/社招、中英文等），动态生成面试脚本（题目列表、追问策略）。
- 面试房间（文字流）：实现与 AI 面试官的多轮对话、追问触发、进度可视化及中途退出/生成报告功能（MVP 核心为文字模式，音视频做占位或基础支持）。
- 结构化报告模块：根据面试记录由 AI 生成多维度的评估报告（含打分、亮点、风险点证据引用、下一步计划）。
- 复盘中心模块：展示历史面试记录及高频薄弱项（错题本）。
- 额度与安全控制：实现免费用户的每日次数限制和基础的输入过滤。

## Impact
- Affected specs: 全新项目，从零到一实现完整 PRD 需求。
- Affected code: 全局项目结构构建、前后端路由、数据库 Schema 设计、AI 提示词工程构建。

## ADDED Requirements
### Requirement: 100% 还原 UI 设计
- **系统要求**：所有的前端页面和组件必须严格基于提供的 `index.html` 中的内联 CSS 进行提取和复用。禁止自由发挥修改间距、色彩、字体等细节，必须像素级还原。

### Requirement: TS 全栈开发架构
- **系统要求**：项目要求端到端类型安全，前后端统一使用 TypeScript。建议采用 Next.js App Router 模式，配合 Prisma 作为 ORM 访问关系型数据库。

### Requirement: AI 面试编排引擎
- **系统要求**：系统需在后台与 AI 大模型交互，需设计结构化的 Prompt 链（解析简历JD -> 生成面试大纲 -> 对话与智能追问 -> 生成包含证据引用的总结报告），确保输出的 JSON 格式稳定且响应迅速。
