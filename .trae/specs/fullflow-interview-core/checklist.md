# 验收清单 (Checklist)

- [x] 全流程列表页能展示真实的当前轮次、下一轮时间、通过/淘汰状态和缺席风险
- [ ] 全流程正式轮次默认进入语音 + 视频房间，媒体失败时能显式降级
- [ ] AI 回答在文字区表现为真实流式输出，而不是完整回复后的拆句伪流式
- [ ] AI 语音播报、实时字幕和文字流式内容保持一致
- [x] 点击 `采集公开面经` 后会创建真实任务、展示进度，并将公开网页审核后的洞察写入数据库
- [x] 提问链与评审链已拆分为两组多 Agent，互不混用职责
- [x] 主动结束当前轮次后，计划会按淘汰终态收口，不再停留在半截状态
- [x] 腾讯 Java 后端等具体公司岗位场景会生成贴合公司岗位语境的问题，不再出现泛化“综合面试”题干
- [x] 一面、二面、三面、HR 面使用独立 Agent 团，轮次之间能继承上一轮反馈
- [x] 每一轮结束后都会运行 3 个独立评审 Agent，多数票过半才判定本轮通过
- [ ] 审核员 Agent 能输出本轮通过、淘汰或待验证结论，并影响下一轮安排
- [x] 评审团投票结果、裁决摘要和改进建议能沉淀到复盘中心可消费结构
- [ ] 候选人前序表现达标时会触发限时算法题，且算法题得分参与最终评分
- [ ] 候选人中途退出、关闭页面、刷新页面后，系统会判定淘汰且不允许继续当前轮次
- [x] 候选人超过约定时间 10 分钟未参加下一轮时，系统会判定缺席淘汰
- [ ] 系统能发送邮件 + 站内信，覆盖下一轮邀约、提醒、淘汰和缺席通知
- [x] 每一轮面试结束后都有结构化反馈，候选人可查看复盘与改进建议
- [ ] 本轮改造不会破坏阶段面试、模板体系、画像页和既有复盘中心主链路
