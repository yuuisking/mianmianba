# Tasks
- [ ] Task 1: 固化学习中心题库内容质量专项的边界、目标与优先级。
  - [ ] SubTask 1.1: 明确本专项以“内容质量”为唯一主目标，不与 UI 样式优化混杂推进。
  - [ ] SubTask 1.2: 列出第一批重点治理题库，并给出优先顺序与抽检范围。
  - [ ] SubTask 1.3: 明确本专项的最终验收标准是“能学、能练、能答、能卖”，而不是只看有无内容。

- [ ] Task 2: 为重点题库建立知识蓝图与覆盖矩阵。
  - [ ] SubTask 2.1: 梳理 Java 集合、Java 并发、JVM、MySQL、Redis、Spring 等重点题库的必修知识树。
  - [ ] SubTask 2.2: 为每个题库定义“核心概念 / 原理机制 / 使用场景 / 对比辨析 / 高频追问 / 实战易错点”的覆盖骨架。
  - [ ] SubTask 2.3: 校准题目顺序与关联关系，确保同库内部具备连续学习路径。

- [ ] Task 3: 升级题目文档的内容质量标准，而不止是章节结构标准。
  - [ ] SubTask 3.1: 为“知识点总结”定义必备信息项、最低信息密度和禁止出现的低质量模式。
  - [ ] SubTask 3.2: 为“面试常问”定义真实面试表达、追问方式和回答路径要求。
  - [ ] SubTask 3.3: 为“参考答案和解析”定义标准回答骨架，包括结论、原理、场景、边界和误区。

- [ ] Task 4: 强化 Agent 生成链路的内容治理能力。
  - [ ] SubTask 4.1: 改造 Agent 提示词，使其围绕题库蓝图和内容骨架生成，而不是自由发挥。
  - [ ] SubTask 4.2: 对跑题、重复、空泛、套话、信息不足等内容建立自动识别与重试规则。
  - [ ] SubTask 4.3: 约束知识点模式生成连续知识文档，约束面试模式生成可直接复述的问答内容。

- [ ] Task 5: 升级后台评分、预警与发布门禁。
  - [ ] SubTask 5.1: 在现有结构评分基础上加入主题纯度、知识密度、面试可用性和语言质量评分。
  - [ ] SubTask 5.2: 在后台结果页展示明确的低质量原因，支持运营或审核快速定位返工点。
  - [ ] SubTask 5.3: 对重点题库启用更严格的发布门槛，不满足标准不得进入正式发布态。

- [ ] Task 6: 开展第一批重点题库的批量提质与抽检。
  - [ ] SubTask 6.1: 按优先级完成第一批重点题库的蓝图校准、题目重写与质量复核。
  - [ ] SubTask 6.2: 对每个重点题库抽检 8 到 12 题，覆盖知识点模式、面试模式、答案和关联题。
  - [ ] SubTask 6.3: 汇总抽检问题并进行第二轮返工，直到重点题库整体质量达标。

- [ ] Task 7: 建立专项验收闭环与上线策略。
  - [ ] SubTask 7.1: 形成专项验收清单，明确每类题库、每类题目、每类场景的检查方法。
  - [ ] SubTask 7.2: 确定第一批重点题库的上线策略与回滚策略，避免低质量内容误发布。
  - [ ] SubTask 7.3: 完成线上验收记录，沉淀为后续全量题库治理模板。
