# 学习中心题库内容质量专项 Spec

## 背景
当前学习中心的性能问题已经完成一轮重点治理，接下来产品的核心矛盾不再是“能不能打开”，而是“打开之后能不能学到东西、能不能拿去面试、能不能形成持续学习价值”。用户已经明确反馈，当前题库内容的主要问题不是个别题目瑕疵，而是整体质量不达标，因此需要把“题库内容质量”从零散修补升级为单独专项治理。

## 目标
1. 将学习中心从“能看”提升为“能学、能练、能答、能卖”的题库产品。
2. 为所有正式题库建立统一的内容质量标准，避免继续出现跑题、空泛、拼接感强、知识密度低的问题。
3. 为每个题库建立按主题拆分、按题目抽检、按文档评分、按发布拦截的治理闭环。
4. 优先把用户最关注、最容易验收的正式题库做深做透，再向其他题库批量推广。

## 非目标
1. 本专项不重新设计学习中心前台 UI 结构，除非质量治理需要极少量辅助展示调整。
2. 本专项不重做后台多 Agent 整体架构，只在现有链路上强化内容质量生产、审核、重试和发布门禁。
3. 本专项不以“继续扩题量”为唯一目标，只有在质量达标的前提下才允许扩题。

## 当前核心问题
1. 文档有结构但缺少有效信息，用户读完仍无法掌握概念、边界和答法。
2. 不同题库之间质量波动大，同一题库内部也存在“有的题能学、有的题很水”的情况。
3. 面试模式下的问题、追问和答案仍不够贴近真实面试表达。
4. 知识点模式虽然已经脱离题目换皮，但知识组织仍不够成体系，缺少“为什么、适用场景、易错点、对比项”。
5. 后台虽然已有结构校验和基础质量分，但离真正可售卖的学习内容还有明显差距。

## 质量定义
所有正式题目文档必须同时满足以下六个质量维度：

### 1. 主题相关性
- 题目、知识点、追问、答案必须严格围绕当前题库主题，不允许大领域串味。
- 例如“Java 集合”题库中的内容必须聚焦集合体系、使用场景、复杂度、并发安全、源码机制和常见误区，不允许泛化成 Java 全家桶。

### 2. 知识密度
- 每篇知识点文档必须让用户看完后能明确回答“这是什么、怎么用、什么时候用、为什么这样设计、常见坑是什么、和相近概念有什么区别”。
- 严禁只有泛泛定义、无场景、无边界、无对比、无结论。

### 3. 面试可用性
- 面试常问必须接近真实面试官表达，而不是模板化标题拼接。
- 参考答案和解析必须支持用户直接复述，能够形成“结论 -> 原理 -> 场景 -> 风险/边界”的回答链路。

### 4. 学习连续性
- 同一题库内部的题目顺序、知识递进、关联题跳转必须支持连续学习。
- 不允许题目分布过散、前后跳跃过大、同类知识点重复堆砌。

### 5. 语言质量
- 文案必须专业、自然、简洁，不要有 AI 拼接感、废话、重复句式、机械套话。
- 标题、段落、列表和结论必须易读，不允许出现明显口水文。

### 6. 发布门禁
- 任何不满足最低质量标准的题目，不得进入正式发布态。
- 质量治理必须先于发布，不允许“先上线再补质量”成为常态。

## 质量治理范围

### 题库优先级
第一批优先治理以下高价值题库：
1. Java 集合
2. Java 并发
3. JVM 基础
4. MySQL 基础
5. Redis 基础
6. Spring 基础
7. SpringBoot 基础
8. 计算机网络基础

第二批再覆盖：
1. SpringCloud 基础
2. Kafka 基础
3. 操作系统基础
4. 设计模式基础
5. JavaScript 基础
6. 前端基础与浏览器相关题库

## 治理策略

### 第一阶段：立标准
1. 把“知识点总结 / 面试常问 / 参考答案和解析”从结构完整，升级到内容完整。
2. 为每个板块定义最低信息密度、必要信息项和禁止出现的低质量模式。
3. 形成题目级、题库级、批量发布级三层质量评分口径。

### 第二阶段：建蓝图
1. 为每个重点题库建立标准知识蓝图，明确必须覆盖的主题簇、知识路径和高频题型。
2. 每个题库都要有“核心概念 -> 使用方式 -> 原理机制 -> 对比辨析 -> 场景实战 -> 高频追问”的内容骨架。
3. 扩题必须建立在题库蓝图之上，禁止再做无规划堆题。

### 第三阶段：强生成
1. 强化 Agent 生成提示词和结构化产物约束，不再只要求章节齐全，而是要求章节有效。
2. 对知识点模式文档要求连续讲解，对面试模式要求可直接拿来答。
3. 对明显水文、跑题、重复、套话内容执行自动重试或退回。

### 第四阶段：强审核
1. 增加可学习性、信息密度、面试表达质量、主题纯度等质量打分。
2. 后台结果页必须能直观看到哪些题需要返工，为什么返工。
3. 对重点题库增加抽检清单，确保不是只靠模型分数自证。

### 第五阶段：强验收
1. 每个重点题库抽查至少 8 到 12 题。
2. 每个抽查样本都要覆盖知识点模式、面试模式、答案质量、关联题连续性四个方向。
3. 线上验收以“用户能否直接学会并复述”为最终标准，而不是“页面上有内容”。

## 产出要求
本专项最终必须产出以下成果：
1. 重点题库内容质量标准文档。
2. 重点题库知识蓝图与覆盖矩阵。
3. 后台质量评分与发布门禁升级方案。
4. 第一批重点题库的批量提质结果。
5. 可执行的抽检清单与最终验收记录。

## 实施原则
1. 先重点题库，后全量铺开。
2. 先题库蓝图，后批量生成。
3. 先质量门禁，后继续扩题。
4. 先抽检闭环，后批量上线。
5. 坚决不允许为了赶进度牺牲内容质量。

## 验收标准
1. 抽查重点题库时，不再出现明显跑题、空泛、无知识密度的题目。
2. 知识点模式能让用户学懂知识，不再只是答案换皮。
3. 面试模式的问题、追问和答案明显更接近真实面试表达。
4. 同一题库内部的题目顺序、递进和关联阅读更自然。
5. 后台对低质量内容有明确可见的拦截与返工机制。
6. 用户主观体验上，能够明确感知“现在的题库内容终于能学到东西了”。
