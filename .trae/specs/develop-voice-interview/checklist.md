# 验收清单 (Checklist)

- [x] “发起面试”选择语音模式后，能进入 `/interview?mode=voice` 且页面可用（桌面 Chrome）。
- [x] 语音面试界面足够简单：只包含必要信息（状态、字幕、说话/停止按钮），无花哨装饰。
- [x] 浏览器自动播放限制 (Autoplay Policy) 已处理：用户首次交互后成功解锁音频上下文，不会发生首次 TTS 播放无声的问题。
- [x] AI 输出过程中字幕为实时流式更新，并自动滚动到最新内容。
- [x] 用户按住说话录音，松开后能完成 ASR，并将识别文本作为用户消息进入对话。
- [x] AI 响应能被 TTS 播报，音色自然，不是明显“机器人音”。
- [x] 用户可以随时打断 AI：开始说话/点击停止时，AI 音频立刻停止，且不会继续播放后续段落。
- [x] 错误处理与降级：
  - [x] 当 TTS/ASR/供应商接口失败时，页面有清晰的中文错误提示且不泄露密钥/内部信息。
  - [x] ASR 多次失败或网络高延迟时，UI 提示用户并提供降级到“文字面试”的入口。
  - [x] TTS 个别片段失败或超时时，不会阻断整个字幕渲染和面试流程，能够实现优雅降级。
- [x] 火山/豆包相关密钥仅存在于服务端环境变量中，客户端不可见。
- [x] 复盘中心、专项训练、文字面试等现有功能不受影响（回归通过）。
