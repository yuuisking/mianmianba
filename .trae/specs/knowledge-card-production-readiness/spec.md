# 知识点速通卡片量产专项 Spec

## 背景
当前学习中心已经验证了“学习文档 + 面试训练”的基础产品方向，但现阶段仍只达到“小规模试生产”标准，距离“可稳定量产并全量铺开”还有明显差距。新的“知识点速通卡片”不是对现有题库简单加一层内容，而是要把面面吧从“面试突击工具”升级为“知识学习 + 面试训练”双轮驱动平台。

现阶段有三个并列且必须一起治理的问题：
1. 内容质量还没有形成稳定量产标准，缺少统一的知识卡片 contract、来源可信度标准、双向链接规则和发布门禁。
2. 页面样式和信息承载方式还不适合知识卡片量产，尤其是文档详情页中间正文区过窄，屏幕可用于展示知识内容的区域太小，无法支撑“长文档深读 + 代码示例 + 对比表格 + 练习反馈”的阅读体验。
3. 机制链路类知识点经常需要流程图辅助理解，但当前专项还没有把“是否需要流程图”以及“流程图展示是否完整”定义为显式质量门禁。

因此，本专项不直接进入量产，而是先完成“量产就绪化”：建立知识卡片的质量门禁、页面母版、量产流程和验收闭环，再进入试生产。

## 目标
1. 为知识点速通卡片建立可量产的内容标准，确保卡片不是“换个壳子的面试题解释”，而是真正可学习、可复述、可跳转到面试的知识产品。
2. 为知识点卡片建立统一的前台页面母版，确保首页列表、专题列表、详情页、双向链接区、学习路径区和练习区在视觉和交互上形成一套稳定体验。
3. 明确“内容区太窄”的样式问题为 P0 页面体验问题，要求卡片详情页正文区显著放宽，提升屏幕利用率与知识承载能力。
4. 建立从选题、生成、审核、抽检到发布的量产闭环，保证后续可按技术方向和专题逐步扩充，而不是一次性无门槛堆量。
5. 在正式量产前先完成 20-30 张高频知识卡片试生产，并通过 ECS 真实页面验收。

## 今晚重规划范围（2026-05-10）
本轮不继续扩展数据库模型、不新增后台配置面板，也不启动“互联网所有知识”的全量量产。为了保证今晚可以整包交付并在明早直接验收，执行范围压缩为一个可上线的知识卡片 MVP 闭环：

1. 首页必须保留 `题库 / 知识卡片 / 学习路径` 三类入口，但不能继续把三类内容整页混排；必须通过 sheet / 分页式切换保证任一时刻只展示一个主面板。
2. 首页知识卡片与学习路径一律从现有已发布文档规则派生，不引入新表、不做迁移。
3. 详情页必须补齐四类稳定关联模块：`前置知识 / 相关知识点 / 相关面试题 / 学习路径建议`。
4. 详情页继续沿用“连续知识文档”阅读形态，不能因为新增关联模块而把正文重新压回窄列。
5. 所有新增能力必须直接部署到 ECS，并以公网浏览器真验为准，不以本地无库环境代替。
6. “面面吧智能助手”原有链路不得改动，只允许保留现有入口和上下文收敛行为。

## 非目标
1. 本专项不在第一阶段直接实现“互联网所有知识”的全量覆盖。
2. 本专项不在第一阶段上线个性化推荐算法、社区功能或企业版能力。
3. 本专项不重做整个学习中心架构，只在现有学习中心和文档系统之上扩展知识卡片能力。
4. 本专项不在没有质量门禁和样式母版的情况下直接开始 50 张以上批量生产。

## 当前核心问题
### 1. 内容侧
1. 现有学习内容质量更接近“重点题库试生产”，还没有抽象出适合所有技术方向的知识卡片标准。
2. BRD/PRD 已定义了功能结构，但还没有把“哪些内容才能进入量产”“什么情况必须拦截”写成执行级标准。
3. “来源与可信度”“易混淆点”“练习反馈”“面试会怎么考”等模块虽然方向明确，但还没有形成统一发布门槛。
4. 如果直接量产，极易复制出现有问题：内容密度波动、专题串味、知识卡片与面试题链接松散、题感强于知识感。

### 2. 样式与体验侧
1. 当前整片文档阅读区，尤其是中间正文区，整体偏窄，长知识内容在大屏下利用率不足。
2. 代码示例、对比表格、长说明块、练习反馈块在当前宽度下容易显得拥挤，影响“系统学习”体验。
3. 现有学习中心主要围绕题库型文档设计，而知识点卡片需要更强调“连续知识阅读”和“关联跳转”。
4. 如果不先定义页面母版，后续会出现卡片详情页、题库详情页、学习路径页各自长得不一样，样式持续漂移。

## 质量定义
所有进入正式发布态的知识点卡片必须同时满足以下八个维度：

### 1. 知识完整性
- 必须能回答“这是什么、为什么重要、怎么用、什么时候用、常见误区是什么、和相近概念有什么区别”。
- 不能只有定义和一段口语化解释。

### 2. 信息密度
- 5 分钟速读必须能让用户抓住核心概念和应用价值。
- 15 分钟深读必须能承载分节讲解、对比分析、代码示例或机制解释。
- 不允许水文、重复句式、模板化废话。

### 3. 真实场景性
- 必须给出具体且真实的应用场景，不允许用通用模版句替代场景。
- 场景要能对应用户角色：校招学习、社招提升、转行补基础、在职碎片学习。

### 4. 面试可迁移性
- 每张卡片都必须能明确告诉用户“这个知识点在面试里会怎么考”。
- 相关面试题必须体现直接关联、间接关联或延伸关联。

### 5. 练习可验证性
- 每张卡片要支持最小练习闭环，至少定义练习类型、答案、解析和错误后的回看路径。
- 练习反馈必须能帮助用户确认“学会了没有”，而不是只给对错。

### 6. 来源可信度
- 每张卡片必须标注来源与可信度，不允许没有出处。
- 对关键机制类内容，必须优先引用官方文档、语言/框架官方说明或公认权威资料。

### 7. 语言与结构质量
- 语言必须专业、自然、可复述。
- 章节结构要统一，但不能出现机械拼模板感。

### 8. 图解完整性
- 不是所有知识卡片都必须有流程图，但凡是涉及执行链路、状态流转、组件协作、时序关系、生命周期或机制分步过程的卡片，必须评估是否需要流程图。
- 对“需要流程图”的卡片，流程图属于正式内容的一部分，不能缺失、不能以一句纯文字占位替代。
- 流程图必须完整展示：不截断、不被容器裁切、不因长文本节点而信息缺失、不在打印或导出时断成无法阅读的碎片。
- 流程图中的节点文案必须短词化，详细解释放图下说明区，不允许把整段解释硬塞进节点导致图表失真。

### 9. 发布门禁
- 任何缺少关键模块、来源不清、信息密度不达标、双向链接缺失的卡片，不得进入正式发布态。
- 对应需要流程图的卡片，如果流程图缺失、展示不完整、渲染失败或无法阅读，也不得进入正式发布态。

## 页面与样式母版要求
### 1. 首页与列表页
1. 学习中心首页需要明确“面试题库 / 知识点卡片 / 学习路径”三类对象的层级关系。
2. 知识点卡片列表不能只是缩小版题库卡片，要突出“预计时长、掌握状态、练习数、关联面试题数”。
3. 支持技术方向筛选和内容类型切换，但首屏不能过度拥挤。

### 2. 卡片详情页
1. 必须沿用“连续知识文档”思路，而不是重新回到多卡片工作台感。
2. 中间正文区必须显著放宽，不能再让主要知识内容只占很窄一列。
3. 页面宽度策略要支持大屏阅读：在桌面宽屏下，正文区应成为视觉主角，而不是被左右栏挤压。
4. 代码块、对比表格、图解、练习反馈块都必须在新宽度下保持舒展，不拥挤、不频繁折行。
5. 左侧章节列表和右侧目录都必须为正文服务，不能反过来压缩正文。

### 3.1 流程图专项要求
1. 对涉及流程、机制、时序或状态流转的知识卡片，详情页必须支持稳定图解展示。
2. 图解必须优先采用稳定、可控的结构化图组件，而不是依赖自由生成且不可控的复杂图。
3. 图中只保留短节点，图下必须提供“图中节点说明”或“补充说明”。
4. 流程图必须支持放大查看、横向容错和完整阅读，不能出现关键节点被截断、箭头链路缺失或内容超出容器不可见。

### 4. 正文区宽度专项要求
1. 当前“整片文档太窄、尤其是内容部分太小”被定义为 P0 页面体验问题。
2. Spec 实现阶段必须重新设计阅读容器宽度、左右栏占比和断点策略，而不是只微调一个 `max-width`。
3. 验收时必须在 1440px、1728px、1920px 三档桌面宽度下核对正文实际可视面积。
4. 验收标准不是“看起来宽了一点”，而是：
   - 正文能明显承载长段落、表格和代码块
   - 大屏下左右留白显著减少
   - 用户主观感受不再是“中间只有一条窄内容带”
   - 流程图、图下注释和代码块不会因为宽度不足被挤压到难以阅读

### 5. 双向链接区
1. 面试题详情页中的“前置知识”需要卡片化，但不能抢掉正文阅读主线。
2. 知识卡片详情页中的“面试会怎么考”“相关知识点”要成为稳定模块，不可选配化。
3. 双向链接模块要有明确标签与关系说明，不只是若干裸链接。

### 6. 学习路径区
1. MVP 阶段只做预设学习路径，不做复杂个性化算法。
2. 路径页要能展示阶段、完成状态、下一步学习建议，但不能做成任务管理工具。

## 量产流程要求
### 第一阶段：立标准
1. 固化知识卡片 contract：结构、来源、双向链接、练习、状态字段、质量分维度。
2. 固化页面母版：首页、列表页、详情页、学习路径页的布局和模块标准。
3. 固化试生产门禁：不满足门禁不得进入批量生产。

### 第二阶段：做样板
1. 先选 20-30 张高频知识卡片做试生产。
2. 技术范围优先从通用基础、Java、前端、Python 中选高频且最能验证链路的卡片。
3. 每个方向都要至少有一张包含代码示例、一张包含对比辨析、一张包含练习反馈。

### 第三阶段：跑审核
1. 用规则质检 + AI 质检双层检查知识卡片。
2. 每张卡片都要评估结构完整性、来源质量、知识密度、练习闭环、双向链接完整性，以及“是否需要流程图 / 流程图是否完整展示”。
3. 未通过卡片直接回退，不得混入试生产集。

### 第四阶段：做验收
1. 双窗口验收必须覆盖内容、样式、跳转、练习、进度、来源五类检查。
2. 至少抽查桌面宽屏阅读体验，验证“内容区过窄”问题是否被真正解决。
3. 通过后才允许从 20-30 张扩到 50 张以上。

## 数据与内容治理要求
1. 知识卡片应拥有独立类型标识，不能继续和面试题在数据语义上混为一谈。
2. 卡片与面试题、卡片与卡片之间的关联关系必须可配置、可审核、可展示。
3. 要保留卡片学习进度、阅读完成度、练习结果等基础行为数据。
4. 根目录与 `doc/` 下存在同名 PRD 文件，属于文档源分叉风险；实现前必须收口为单一权威版本。

## 实施原则
1. 先专项 spec，后功能开发。
2. 先质量门禁，后样式打磨完成度。
3. 先页面母版，后批量生产。
4. 先 20-30 张试生产，后 50 张以上扩量。
5. 先通过双窗口验收，后进入批量发布。

## 验收标准
1. 已形成独立的知识点卡片专项规范，质量、样式、量产流程三条线都被纳入。
2. 页面规范中已明确“正文区过窄”为 P0 问题，并定义了实现与验收标准。
3. 已定义知识卡片的最小发布门禁，包括结构、来源、练习、双向链接、流程图完整性和质量评分。
4. 已定义 20-30 张试生产范围，而不是直接进入全量量产。
5. 首页可看到 `题库 / 知识卡片 / 学习路径` 三类入口，并通过 sheet 切换在同一个学习入口内分屏查看，避免三类内容同页堆叠。
6. 详情页可稳定看到 `前置知识 / 相关知识点 / 相关面试题 / 学习路径建议` 四类模块，且每类模块都带明确关系标签或行动入口。
7. 宽屏下正文区仍保持显著主区域，新增关联模块不会把阅读区重新挤窄。
8. 公网 ECS 页面实测通过，不依赖本地无库环境做“理论验收”。

2. 已将学习中心首页改为保留三类入口，但只展示一个当前 sheet 面板；当前可在 `知识卡片 / 学习路径 / 题库列表` 之间切换，并保留原搜索与分类筛选。
3. 已将题目详情页补齐四类关联模块，并在真实线上文档中验证：
