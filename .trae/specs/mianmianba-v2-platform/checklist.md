# 面面吧 v2.0 平台迭代 Checklist

## 范围边界
- [ ] 当前阶段已明确冻结学习中心主线，不把学习中心重做混入 v2.0 第一阶段。
- [ ] 当前阶段已明确首页、发起面试、专项训练、复盘中心、代码面试间为主范围。
- [ ] 当前阶段没有把会员管理整体重做混入主链路。

## 首页
- [ ] 首页视觉和模块结构与 v2.0 demo 高度一致。
- [ ] 首页展示真实进度，而不是长期静态写死数据。
- [ ] 首页存在“继续训练”能力，并能跳转到正确上下文。
- [ ] 首页存在 Top 3 薄弱点，并可联动复盘或专项训练。

## 发起面试
- [ ] 发起面试入口先区分 `阶段面试` 与 `全流程面试`。
- [ ] 阶段面试链路支持画像确认，而不是直接硬进面试房间。
- [ ] 全流程面试链路支持目标公司、岗位、职级、JD、简历与多轮计划。
- [ ] 发起面试不再只是单场练习工具，而是进入真实招聘模拟的入口。

## Agent 评审团
- [ ] 已明确 Planner / Interviewer / Evidence / Scoring / Summary / Report / Coach 等角色边界。
- [ ] 面试问题不是纯随机生成，而是结合岗位、JD、简历、轮次和历史弱项。
- [ ] 评分、总结和报告不是同一个 Agent 粗暴一次性完成。

## 代码面试
- [ ] 所有合适的面试流程都支持代码题环节。
- [ ] 代码间具备题干区、编辑区、辅助区、运行区与提交区。
- [ ] 系统能记录代码编辑、运行、提交和提示使用情况。

## 复盘中心
- [ ] 复盘中心能明确看出“哪里有问题”，而不只是看一个总分。
- [ ] 复盘中心包含高频薄弱点、历史记录、推荐补强计划。
- [ ] 复盘中心可以回流到专项训练或后续面试计划。

## 数据底座
- [ ] 已设计用户成长快照、薄弱点、面试计划、轮次、题目、回答、代码、评分、报告等实体。
- [ ] 数据模型支持后续做用户画像、趋势分析和成长档案导出。
- [ ] 系统保留过程数据，而不是只保留最终总分。

## 状态机
- [ ] 面试计划状态清晰可追踪。
- [ ] 轮次状态、问题推进状态、代码状态清晰可追踪。
- [ ] 打断追问、切代码题、结束面试、生成报告都有明确状态迁移。

## 迭代策略
- [ ] v2.0 第一阶段优先打稳平台骨架、数据底座和关键页面。
- [ ] 多模态高级能力采用渐进增强，而不是一次性堆满。
