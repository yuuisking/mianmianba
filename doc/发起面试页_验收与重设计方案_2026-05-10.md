## 2. 问题判断

### 2.1 布局为什么难受

当前页面最大的问题不是模式选择错了，而是：

> 模式之后的配置布局没有节奏，所有配置几乎都在同一个视觉层级里抢注意力。

具体表现：

- 左侧一次性塞入岗位、公司、职级、方式、强度、重点、模板、结束方式
- 同一类配置大量使用平铺按钮，导致界面非常吵
- 一级决策、二级决策、补充配置、高级配置没有被明显区分
- 右侧虽然只保留 JD 和简历，但左侧配置过重，整页仍然显得很堵

所以现在页面给人的感受不是“清爽”，而是：

- 信息很多
- 控件很多
- 每个控件都在抢位置
- 用户不知道现在最该做哪一步

### 2.2 确认配置弹窗为什么乱

当前弹窗代码在 `src/app/setup/page.tsx` 中是从 [page.tsx:L1709-L1800](file:///Users/yangyu/Desktop/resumer/src/app/setup/page.tsx#L1709-L1800) 渲染出来的。

它的问题不是“完全没内容”，而是内容组织方式很乱：

- 左边是面试官形象图和几行风格说明
- 右边是摘要字段
- 下面再放按钮
- 但真正对用户有价值的信息没有被分成“摘要 / 下一步 / 风险 / 确认动作”

当前弹窗最明显的混乱点：

- 一张形象图占了很强视觉权重，但不是真正最重要的信息
- 文案“确认后先进入画像解析，再继续到本场面试；文字、语音、视频模式都会保留”信息价值偏低
- 摘要字段是平铺罗列，不够像“临门一脚前的确认页”
- 没有明确告诉用户“确认之后会先发生什么”
- 没有把关键配置、下一步流程、当前模式策略分成不同区块

### 2.3 当前页面最本质的问题

我现在的判断比上一轮更收口：

> 页面不是“模式入口错了”，而是“模式之后的排布错了”；弹窗不是“缺少字段”，而是“字段结构和视觉主次错了”。

## 3. 我建议的重设计方向

这次方向直接收口成一句话：

> 左边只放配置，右边只放上传简历，别再加别的。

具体要求：

1. 保留 `阶段面试 / 全流程面试` 模式切换
2. 页面主体严格拆成左右两栏
3. 左边只放配置
4. 右边只放上传简历 / 粘贴简历
5. 不再加入流程说明卡、额外解释卡、分析卡、辅助模块
6. 确认配置弹窗只做摘要确认，不再做复杂拼贴

---

## 4. 建议的新信息架构

我建议把 `setup` 页收口成下面这套结构：

### 4.1 页面结构

1. 页头区
2. 模式切换区
3. 左侧配置区
4. 右侧上传简历区
5. 底部确认动作

### 4.2 各区块职责

#### 页头区

只回答两件事：

- 当前页面是“发起面试”
- 你现在是在配置阶段面试还是全流程面试

不要再堆说明文案。

#### 模式切换区

保留：

- `阶段面试`
- `全流程面试`

但只作为顶层模式切换，不把模式说明写得太满。

#### 左侧配置区

左侧只放：

- 目标岗位
- 目标公司
- 目标职级
- 面试方式
- 面试强度
- 自动结束方式
- 本场重点

#### 右侧上传简历区

右侧只放：

- 上传简历
- 粘贴简历
- 识别结果反馈

#### 底部确认动作

底部只保留：

- 返回修改
- 确认配置 / 确认并开始

## 6. 关键交互

### 6.1 模式切换

- 保留顶部 `阶段面试 / 全流程面试`
- 切换模式后，页面主体区域整体替换
- 不要一个模式过重、另一个模式过空

### 6.2 左配右传

- 左边只负责配置
- 右边只负责简历上传和粘贴
- 两边职责不能混

### 6.3 自动识别岗位

- 继续保留真实简历驱动的自动识别
- 识别成功后直接回填岗位
- 识别结果以轻提示展示，不要让提示文本抢版面

### 6.4 确认配置弹窗

确认弹窗必须改成极简摘要结构：

1. 本场摘要
2. 下一步说明
3. 操作按钮

不要再加入大面积形象图、额外说明块和多余信息区。

### 6.5 确认动作

弹窗内只保留两个核心动作：

- 返回修改
- 确认并开始 / 确认并预约

并且按钮上方必须把“确认后的流程”解释清楚。

## 9. 视觉风格建议

### 9.1 基调

- 清浅底色
- 柔和边框
- 强留白
- 控件轻量
- 强调“卡片层级”，不强调“按钮数量”

### 9.2 控件风格

- 少用平铺大按钮
- 多用分组卡、轻量 chip、下拉、单选卡
- 选中态明显，未选态降噪
- 同一层级控件视觉权重保持一致

### 9.3 布局风格

- 页面横向分区必须非常清楚
- 左边就是配置
- 右边就是上传简历
- 不允许再塞第三种信息模块
- 确认弹窗必须像“确认摘要页”，不能像“信息拼贴卡”

### 9.4 文案风格

- 标题短
- 副标题少
- 每个分组只保留一句必要解释
- 去掉重复性说明文案

## 10. 直接给执行模型的改造要求

1. 保留 `阶段面试 / 全流程面试` 双模式，不要推翻当前模式选择思路。
2. 重做模式后的主布局，严格改成“左边配置 / 右边上传简历”。
3. 不允许继续加入流程说明卡、辅助说明卡、分析卡、附加模块。
4. 公司、职级、重点、结束方式的控件必须明显降噪，减少平铺按钮数量。
5. 目标公司联动后的职级展示必须更自然，例如 `T2 · 1-3 年经验`，不要只显示 `T1/T2/T3`。
6. 确认配置弹窗必须升级成“摘要确认层”，只保留摘要、下一步说明和操作按钮。
7. 弹窗里不允许再出现抢视觉的大面积弱价值内容。
8. 页面整体视觉必须遵循“克制、轻量、清爽、层级清楚”的标准。
9. 以 `doc/发起面试页_UI重设计预览.html` 作为布局参考稿。

## 11. 最终结论

这次重新检查后的结论很明确：

> “阶段面试 / 全流程面试”的选择思路本身没有问题，真正的问题是模式后的布局太乱，以及确认配置弹窗没有被设计成一个高质量的确认层。

所以接下来不应该再争论模式本身，而应该集中做两件事：

1. 把主页面改成“左边配置、右边上传简历”
2. 把确认配置弹窗改成“极简摘要确认层”

只要这两件事做好，`setup` 页的清爽感和产品完成度才会真正上来。
